# coding-principles

> 범용 소프트웨어 설계 원칙 가이드 — DRY, SOLID, KISS, YAGNI, 재사용, 공용화, 클린 코드

## When to Use

- 코드 작성, 리팩토링, 구조 설계
- 중복, 재사용, 공용화, 모듈화
- DRY, SOLID, 클린 코드, 코드 리뷰
- OOP, 함수형, 설계 원칙, 의존성

## What It Covers

- 기존 코드 우선 (Reuse Before Create)
- DRY / Rule of Three
- SOLID 원칙
- KISS & YAGNI
- OOP vs FP 상황별 선택 가이드
- 공용화 판단 기준
- 의존성 방향 (Layered/Clean Architecture)
- 네이밍 원칙 (범용)
- 에러 핸들링 원칙 (범용)
- 테스트 원칙 (범용)
- 코드 리뷰 관점

## Relationship to Language-Specific Skills

이 skill은 "왜/무엇을"을 다루고, 언어별 skill은 "어떻게(해당 언어에서)"를 다룸:
- TypeScript/Node.js → `typescript-node`
- Java/Kotlin → (추후 추가)
