# Quick Task Tracker

급하게 들어오는 업무를 **최소한의 마찰로** 기록·조회·완료 처리하는 경량 태스크 관리 스킬.

## 저장 위치

```
~/.kiro/tasks/
├── active.md       # 현재 진행 중/미완료 태스크
└── archive.md      # 완료된 태스크 아카이브 (월별 구분)
```

## 태스크 포맷 (active.md)

```markdown
# Active Tasks

| ID | 업무 | 마감일 | 등록일 | 메모 |
|----|------|--------|--------|------|
| 1 | API 스펙 리뷰 | 2026-07-22 | 2026-07-20 | 김과장 요청 |
| 2 | 로그 수집 파이프라인 설계안 | 2026-07-25 | 2026-07-20 | |
| 3 | 주간보고 작성 | 2026-07-21 | 2026-07-20 | 매주 월요일 |
```

- ID는 active 목록 내에서 1부터 순차 부여 (완료 처리 후 재번호 매김)
- 마감일이 없으면 빈칸 (우선순위 최하위로 취급)
- 메모는 선택사항 (요청자, 간단한 컨텍스트 등)

## 핵심 동작

### 1. 업무 등록

사용자가 업무를 말하면 바로 `active.md`에 추가한다.

**입력 예시:**
- "내일까지 API 스펙 리뷰 해야 해. 김과장 요청"
- "7/25까지 로그 수집 설계안 만들어야 함"
- "주간보고 월요일마다"
- "이거 해야 돼: 배포 스크립트 정리"

**파싱 규칙:**
- 날짜 표현("내일", "이번 주 금요일", "7/25", "다음주 월요일")을 YYYY-MM-DD로 변환
- 마감일이 명시되지 않으면 빈칸으로 두되, "급해", "오늘" 같은 표현이 있으면 오늘 날짜로 설정
- 등록일은 항상 오늘 날짜
- 한 번에 여러 건 등록 가능 ("A 해야 하고, B도 해야 해")

**등록 후 응답:**
간결하게 확인만. 예: "추가했어 ✓ (마감: 7/25)"

### 2. 업무 조회

"오늘 할 일", "남은 업무", "이번 주 뭐 해야 돼" 등의 요청 시:

1. `active.md`를 읽는다
2. 마감일 급한 순으로 정렬 (마감일 없는 것은 맨 뒤)
3. 번호를 1부터 다시 매겨서 보여준다
4. 기한 지난 것은 🔴, 오늘인 것은 🟡, 여유 있는 것은 표시 없음

**출력 예시:**
```
📋 남은 업무 (3건)

🔴 1. 주간보고 작성 — 오늘 (7/21)
🟡 2. API 스펙 리뷰 — 내일 (7/22), 김과장 요청
   3. 로그 수집 파이프라인 설계안 — 7/25
```

**조회 범위:**
- "오늘 할 일" → 마감일이 오늘이거나 이미 지난 것
- "이번 주" → 이번 주 일요일까지 마감인 것
- "남은 업무" / "할 일 보여줘" → 전체 active 목록
- "다음 주" → 다음 주 월~일 마감인 것

### 3. 완료 처리

번호를 지정하면 해당 태스크를 `active.md`에서 제거하고 `archive.md`로 이동.

**입력 예시:**
- "1번 끝났어"
- "1, 3 완료"
- "주간보고 끝났어" (이름으로도 매칭)
- "다 했어" (전부 완료 — 확인 한 번 물어봄)

**완료 처리 후:**
- `active.md`에서 해당 행 제거
- `archive.md`에 완료일과 함께 추가
- 남은 태스크 ID를 1부터 재번호 매김
- 간결하게 응답: "1, 3번 완료 처리 ✓ (남은 업무 2건)"

### 4. 수정/변경

- "2번 마감일 7/28로 변경해줘"
- "3번 메모 추가: 팀장님 cc 필요"

해당 행을 직접 수정.

### 5. 아카이브 조회

- "지난주 뭐 했어?" / "완료된 거 보여줘"
- `archive.md`에서 해당 기간 완료 건을 보여줌

## archive.md 포맷

```markdown
# Task Archive

## 2026-07

| 업무 | 마감일 | 완료일 | 메모 |
|------|--------|--------|------|
| 주간보고 작성 | 2026-07-21 | 2026-07-21 | 매주 월요일 |
| API 스펙 리뷰 | 2026-07-22 | 2026-07-22 | 김과장 요청 |
```

월별로 섹션을 구분하여 기록.

## 동작 규칙

1. **최소 마찰**: 사용자가 한 줄만 던져도 파싱해서 등록할 수 있어야 한다. 되묻기는 정말 애매할 때만.
2. **마감일 우선 정렬**: 항상 마감일 급한 순. 같은 날이면 등록일 빠른 순.
3. **자동 재번호**: 완료 처리 후 남은 목록은 항상 1부터 재번호.
4. **파일 없으면 생성**: `~/.kiro/tasks/` 디렉토리나 파일이 없으면 자동 생성.
5. **간결한 응답**: 등록/완료 시 장황하게 말하지 말고 핵심만. 조회 시에만 리스트 포맷.
6. **날짜 유연 파싱**: "내일", "금요일", "다음주 화", "7/25", "25일" 등 자연어 날짜 인식.
7. **중복 방지**: 같은 내용이 이미 있으면 "이미 등록된 것 같은데?" 확인.
8. **반복 업무**: "매주", "매일" 같은 표현이 있으면 메모에 기록해두고, 완료 후 다음 주기로 자동 재등록 제안.

## 트리거 컨텍스트

이 스킬은 사용자가 다음과 같은 말을 할 때 활성화된다:
- 업무 등록 의도: "~해야 해", "~까지 해야 돼", "업무 추가", "할 일 등록"
- 업무 조회 의도: "오늘 할 일", "남은 업무", "뭐 해야 돼", "할 일 보여줘"
- 완료 의도: "~번 끝났어", "~ 완료", "다 했어"
- 변경 의도: "~번 마감일 변경", "~번 메모 추가"

단, 사용자가 코드 작성이나 기술적 논의 중에 "이거 해야 돼"라고 말하는 것은
태스크 등록이 아니라 작업 지시일 수 있으므로, 맥락을 보고 판단한다.
명확히 "기록해", "등록해", "추가해" 같은 표현이 있거나,
마감일이 언급되면 태스크 등록으로 판단한다.
